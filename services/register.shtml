Registering a Public XMPP Service

Thank you for running a public XMPP service! In order to add your service to the list, please do the following:

  1. Subscribe to the operators@xmpp.org email list by visiting <http://mail.jabber.org/mailman/listinfo/operators> or emailing <mailto:operators-subscribe@xmpp.org>. You must subscribe to the list in order to post, but don't worry: this is a low-volume list and a good place to share experience with other operators of public XMPP services.

  2. Send an email message to the list with a subject of "public XMPP service: [yourdomain.tld]". Please provide the following information:

    • The domain name of the XMPP service.
    • The website where users can find more information.
    • The year when the service was launched.
    • The country, latitude, and longitude of the physical machine that hosts the service.
    • The CA that issued the security certificate you use for encryption.
    • The server software used to run the service.
    • The name and JabberID of the primary admin.
    • A brief description of the service.

To make your life easier, a template for your message is provided below, using the information from the jabber.org service (replace data as necessary!).

Please add my public XMPP service to the list at <http://xmpp.org/services/>. The service information is as follows:

  • domain: [jabber.org]
  • website: [http://www.jabber.org]
  • year launched: [1999]
  • country: [USA]
  • CA: [XMPP ICA]
  • server software: [ejabberd]
  • admin name: [Peter Saint-Andre]
  • admin JID: [stpeter@jabber.org]
  • description: [the original Jabber service]

Before adding your service to the list, we typically complete all of the following tasks:

  • Verify that there is indeed an XMPP service running at the domain for server-to-server and client-to-server communications, including the SSL-only port 5223 for ease of certificate checking (via OpenSSL s_client) and backward-compatibility with clients that cannot yet support STARTTLS.
  • Check for appropriate DNS SRV records (_xmpp-client._tcp and _xmpp-server._tcp); these are preferred but not necessary.
  • Communicate with the admin via XMPP.
  • Check the website for accuracy.
  • Validate the certificate against the root cert of the security provider.
  • Verify the request by forwarding it to the email address(es) listed in the whois record for the domain, as well as the hostmaster/postmaster/webmaster addresses.
