XSF Roadmap

Author:Peter Saint-Andre
Version:0.11
Date:2009-03-19

The XSF has identified several high-priority initiatives to help achieve that mission:

  1. Complete definition of Jingle as an open technology for voice and video, file transfer, and generalized session negotiation.

  2. Achieve consensus regarding go-forward file transfer technologies (XEP-0234: Jingle File Transfer).

  3. Incorporate implementation and deployment experience into the core definition of XMPP by completing revisions to rfc3920bis and rfc3921bis.

  4. Define a method for secure, implementable, user-friendly, end-to-end encryption over XMPP (the proposed approach is summarized in draft-meyer-xmpp-e2e-encryption).

  5. Define several optimizations to XMPP's core streaming, presence, and roster protocols for improved performance in mobile environments (mainly XEP-0198: Stream Management and XEP-0237: Roster Versioning).

  6. Complete definition of protocols and best practices for prevention of spam, phishing, denial of service attacks, and abuse on the XMPP network (mainly XEP-0159: Spim-Blocking Control, XEP-0161: Abuse Reporting, and XEP-0165: Best Practices to Discourage JID Mimicking).
