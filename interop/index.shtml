XMPP Interoperability Testing Network

The XMPP Standards Foundation coordinates a private network of XMPP servers that developers can access in order to test their code against a wide range of client and server implementations. This "real-time testing lab" helps the XMPP developer community ensure interoperability among implementations. Read on for further information.

Access Policies

The XMPP Interoperability Testing Network is a private network run exclusively for the benefit of XMPP developers. Access is limited to authorized individuals, who must agree to the Confidentiality Agreement before being accepted (see How to Join). In particular:

  • Server deployments MUST NOT create accounts for anyone except individuals listed under Authorized Participants.
  • Server deployments MUST NOT allow server-to-server access from any domain except those listed under Server Deployments.

How to Join

In order to join the network, you must be an independent XMPP developer or an authorized representative of an approved open-source project or company that builds XMPP software as listed under Approved Projects and Companies.

Before being accepted, you must submit the Confidentiality Agreement and send it to Peter Saint-Andre. The basic intent of the agreement is that you will not publicize any bugs you find in other developers' software, which makes all developers comfortable with exposing their software to interoperability testing.

Please read the entire agreement and make sure you accept its terms. Then initial each printed page, complete the "Acceptance" section at the end of the document, and sign your name. You can submit the agreement in one of the following ways:

  • Scan it to PDF format and email it to Peter Saint-Andre via mailto:stpeter@jabber.org
  • Fax it to +1-303-308-3219 (make sure to include a cover page directing it to Peter Saint-Andre!)
  • Send it via postal mail to XSF, P.O. Box 1641, Denver, CO 80201 USA
